Marketing Lead to Applicant Workflow Process

Overview

This document outlines the process for managing the transition from marketing leads to applicants in Mailchimp, ensuring applicants are excluded from marketing workflows once they submit an application.

The Challenge

Problem: Our Student Information System (SIS) cannot currently integrate directly with Mailchimp.

Solution: Weekly manual import from SIS to Mailchimp to flag applicants and exclude them from marketing workflows.

Simplified Multi-Source Workflow Architecture

Source-Based Segmentation Strategy

Your marketing workflows are now streamlined into source-specific sequences:

Marketing Workflows: Source-Specific ├─ Event Attendees (Manual follow-up → Shared Nurture) │ └─ Manual: Event-specific follow-up email │ └─ Then: Enters shared 6-email nurture sequence │ └─ If already in sequence: Manual email sent + continues current position │ ├─ EOI Form Submitters (Shared Nurture) │ └─ Enters shared 6-email nurture sequence immediately │ ├─ Landing Page Sign-ups (Shared Nurture) │ └─ Enters shared 6-email nurture sequence immediately │ ├─ FindAMasters Enquiries (SPECIAL HANDLING) │ └─ Day 0: Acknowledge source + Privacy notice │ └─ Day 3: Course information │ └─ Day 7: Application process │ └─ Day 14: Deadline reminder │ └─ Monitor: Flag if no engagement after 30 days │ └─ Website Contact Form (Separate Sequence) └─ Day 0: Enquiry response + opt-in confirmation └─ Day 3: Course discovery guide └─ Day 7: Event invitation └─ Day 14: Application information

Shared 6-Email Nurture Sequence:

  • Day 0: Application encouragement
  • Day 3: Open events promotion
  • Day 7: Student stories
  • Day 14: Application deadlines
  • Day 21: Scholarship information
  • Day 30: Reputational comms

All workflows STOP when: Tag "Applicant: [Year]" is added

Enhanced Workflow Conditions

Marketing Workflows (Must Exclude Applicants)

For ALL marketing automation sequences:

Send to contacts who meet ALL of these conditions:

  • Status is "Subscribed"
  • Does NOT have any tag starting with "Applicant:"
  • Does NOT have any tag starting with "Student:"

Optional additional conditions for segmentation:

  • Has tag "Source: [specific source]" (for source-specific campaigns)
  • Has tag "Attended: [event]" (for event follow-up)
  • Country is "[region]" (for regional campaigns)
  • Course Name contains "[programme]" (for course-specific info)

Source-Specific Workflow Setup

Workflow 1: Event Attendee Process

Manual Step (Outside Mailchimp):

  • When: Event concludes
  • Action: Send manual event-specific follow-up email
  • Tag Applied: "Attended: [Event Name] - [Date]"

Automated Step (Mailchimp):

  • Trigger: Tag added "Attended: [Event Name]"
  • Condition: Does NOT have "Applicant:" tag
  • Action:
    • IF contact is NOT already in Shared Nurture Sequence → Add to sequence at Day 0
    • IF contact IS already in Shared Nurture Sequence → Continue from current position
  • Note: Contact receives manual email regardless of sequence status

Workflow 2: Shared Nurture Sequence (EOI Forms + Landing Pages + Event Attendees)

Entry Triggers:

  • Tag added "Source: EOI - [Course Name]", OR
  • Tag added "Source: Landing Page - [Campaign]", OR
  • Tag added "Attended: [Event Name]" (after manual follow-up)

Conditions:

  • Does NOT have "Applicant:" tag
  • Has NOT already completed this sequence

  • Quick wins: "Apply in under 30 minutes"

Day 3: Open Events Promotion

Day 7: Student Stories

  • Current student testimonials

  • Career outcomes

Day 14: Application Deadlines

Day 30: Reputational Communications

  • Rankings and accreditations
  • Industry partnerships
  • Graduate employment statistics
  • Final application encouragement

Workflow ends: When person is tagged as "Applicant: [Year]" or completes Day 30 email

Detailed Process Steps

Stage 1: Marketing Lead (Current Process)

Who they are:

  • Registered for events
  • Expressed interest in courses
  • Consented to marketing emails

Tags they have:

  • Registered: [Event Name]
  • Attended: [Event Name]
  • Source: [Origin]

What they receive:

  • Event-specific follow-up (manual, if attended event)
  • Shared 6-email nurture sequence OR
  • FindAMasters-specific sequence OR
  • Website Contact Form sequence

Goal: Encourage them to submit an application

Stage 2: Application Submitted (Trigger Point)

What happens:

  • Person submits application through your system
  • Application is logged in SIS
  • Data includes: Email, Name, Year of Entry, Application Date

Important: There will be a gap between application submission and Mailchimp update:

  • Maximum 3-4 days (if they apply just after an import)
  • They may receive 1-2 marketing emails after applying
  • Mitigation: Include line in marketing emails: "Already applied? Please ignore this email and await communication from our Admissions team."

Stage 3: Weekly SIS Export & Mailchimp Import

Schedule:

Monday morning (9:00 AM) - Captures weekend + Mon/Tue/Wed applications

Export from SIS:

Required data: Email Address, Year of Entry

Export criteria:

  • All applications submitted since last export
  • Status: "Application Submitted" or equivalent
  • Include: Email address + Year of Entry only

Import to Mailchimp:

Steps:

  1. Save SIS export as CSV
  2. Open Mailchimp → Audience → Import contacts
  3. Upload CSV file
  4. Settings:
    • "Update existing contacts" = ON
    • Status: Keep existing status
  5. Map fields:
    • Email Address → Email Address
    • Year of Entry → Year of Entry (optional - for reference)
  6. Add tag based on year:
    • If Year = 2026 → Tag: Applicant: 2026
    • If Year = 2027 → Tag: Applicant: 2027
    • If Year = 2028 → Tag: Applicant: 2028
  7. Import

Note: You may need to import 3 separate files (one per year) to apply the correct year-specific tag, OR use Mailchimp's conditional tagging if available.

Stage 4: Workflow Exclusion

Once tagged as applicant, they are automatically excluded from marketing workflows and enter the Applicant Journey workflow.

Exclusion + Applicant Journey Workflow (Admissions owned)

Setup:

  1. Marketing workflows automatically stop (due to "Does NOT have Applicant:" condition)
  2. Applicant Journey workflow begins

Trigger: Tag added Applicant: 2026 (create 3 versions for each year)

Email sequence:

a. Day 0 (Immediate): "Application Received - What Happens Next"

  • Confirm application received
  • Outline timeline
  • What to expect

b. Day 7: "Scholarship information" (if applicable)

  • Tips
  • What we look for
  • Common questions

c. Day 14: "Meet Our Current Students"

  • Student testimonials
  • Day-in-the-life content
  • Course highlights

d. Day 21: "Key Dates Reminder"

  • Decision timeline
  • Funding deadlines
  • What to do while waiting

e. Day 30: "Still Have Questions?"

  • FAQ link
  • Contact details
  • Webinar invitation (if applicable)

Workflow ends: When person is tagged as Student: [Year] or after 60 days

Pros:

  • Keeps applicants engaged
  • Automated nurture reduces Admissions workload
  • Consistent experience for all applicants
  • Positions institution positively

Cons:

  • More setup work initially
  • Risk of message conflicts with Admissions
  • Requires content creation
  • Needs coordination with Admissions team

Stage 5: Enrolment & Data Management

When Student Enrols:

Bi-annual process (October and February):

  1. Export from SIS:
    • All students who have accepted offers/enrolled
    • Email addresses only
  2. Import to Mailchimp:
    • Add tag: Student: 2026 (or relevant year)
    • This flags them for deletion
  3. Bi-annual deletion process:
    • Create segment: Has any tag starting with "Student:"
    • Export segment for records (save to your archive)
    • Bulk delete from Mailchimp
    • Reason: They've fulfilled the purpose of marketing consent (became a student)
    • Student Experience team manages all further communications via their system

Important notes:

  • Marketing consent was for recruitment purposes
  • Student communications handled by separate system/team
  • Alumni managed through different audience/process
  • Deletion complies with data minimisation principle (GDPR)

If Application Rejected/Withdrawn:

Decision needed: Do we:

  • Option 1: Remove Applicant: tag, return to marketing pool for future intakes?
  • Option 2: Keep tag, exclude permanently (they already applied once)?
  • Option 3: Add tag Previous Applicant, create special re-engagement workflow?

Recommendation: Discuss with Admissions team. May vary by reason (rejected vs. withdrew).

Tag Architecture Summary

Marketing Stage: Source: EOI - [Course Name] Source: Landing Page - [Campaign] Source: FindAMasters Source: Website Contact Form Registered: [Event Name] - [Date] Attended: [Event Name] - [Date] (Multiple tags can accumulate) Applicant Stage: Applicant: 2026 Applicant: 2027 Applicant: 2028 (Only ONE applicant tag per person) Student Stage (for deletion): Student: 2026 Student: 2027 Student: 2028 (Flags for removal from Mailchimp)

Workflow Exclusion Rules

Marketing Workflows:

Condition: Send to contacts who do NOT have any tags starting with "Applicant:" AND do NOT have any tags starting with "Student:"

Applicant Journey Workflow:

Trigger: Tag added starting with "Applicant:"

Stop condition: Tag added starting with "Student:" OR 60 days elapsed

Monday Morning Import Checklist

Frequency: Monday AM

Step 1: Export from SIS

  • Log into SIS
  • Run report: "Applications Since Last Import"
  • Date range: Since last Monday's import
  • Export fields: Email Address, Year of Entry
  • Save as: SIS_Applicants_[Date].csv

Step 2: Prepare Import File

  • Open CSV in Excel/Google Sheets
  • Check for duplicates (remove if any)
  • Verify email addresses are valid format
  • Confirm Year of Entry values are 2026, 2027, or 2028
  • Count rows (note number of new applicants)

Step 3: Import to Mailchimp - 2026 Applicants

  • Filter CSV for Year of Entry = 2026
  • Save as: MC_Import_2026_[Date].csv
  • Go to Mailchimp → Audience → Import contacts
  • Upload file
  • Settings: "Update existing contacts" = ON
  • Map: Email Address → Email Address
  • Add tag: Applicant: 2026
  • Import
  • Note import summary (X contacts imported/updated)

Step 4: Import to Mailchimp - 2027 Applicants

  • Filter CSV for Year of Entry = 2027
  • Save as: MC_Import_2027_[Date].csv
  • Repeat import process
  • Add tag: Applicant: 2027
  • Import
  • Note import summary

Step 5: Import to Mailchimp - 2028 Applicants

  • Filter CSV for Year of Entry = 2028
  • Save as: MC_Import_2028_[Date].csv
  • Repeat import process
  • Add tag: Applicant: 2028
  • Import
  • Note import summary

Step 6: Verification

  • Go to Audience → Tags
  • Check count for Applicant: 2026 (should have increased)
  • Check count for Applicant: 2027 (should have increased)
  • Check count for Applicant: 2028 (should have increased)
  • Spot check: Search for 2-3 email addresses to confirm tags applied
  • Verify source tags still present (not overwritten)

Step 7: Source Performance Check (Weekly on Mondays)

  • Export applicants from this week (has Applicant tag + this week's date)
  • Count by source tag:
    • How many from Events?
    • How many from EOI Forms?
    • How many from Landing Pages?
    • How many from FindAMasters?
    • How many from Contact Forms?
  • Update Sheet 2: Source Performance Tracking
  • Note any anomalies (e.g., spike from one source)

Step 8: Record Keeping

  • Update tracking spreadsheet Sheet 1 (Weekly Import Log):
    • Date of import
    • Number of new 2026 applicants
    • Number of new 2027 applicants
    • Number of new 2028 applicants
    • Number of students deleted
    • Any issues/notes
  • Update Sheet 2 (Source Performance) if Monday
  • Update Sheet 3 (Source Conversion) if end of month
  • File all CSV exports in archive folder

Step 9: Error Handling

  • Review any import errors from Mailchimp
  • Common issues:
    • Invalid email format → Fix in SIS, re-export
    • Email doesn't exist in Mailchimp → Expected for direct applicants who never attended events (they'll be added)
    • Duplicate emails → Check if person applied for multiple years
  • Document any issues in tracking spreadsheet
